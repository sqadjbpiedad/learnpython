Demo 2: Land Cover Classification

This demonstration utilizes scripts originally developed for the 4-Day Advanced Online Training on AI/ML for Earth Observation for Philippine EO Professionals, part of the CoPhil Programme (EU-Philippines Copernicus Capacity Support Programme), a flagship initiative under the European Union‚Äôs Global Gateway strategy (2025).

Learning Objectives¬∂

  1. Set up and authenticate Google Earth Engine in Google Colab

  2. Acquire and preprocess Sentinel-2 satellite imagery

  3. Calculate spectral indices (NDVI, NDWI, NDBI, EVI)

  4. Create training datasets using verified open data sources

  5. Train a Random Forest classifier

  6. Perform land cover classification

  7. Assess accuracy and generate statistics

Supervised Classification Workflow¬∂

Key Steps¬∂

  1. Preprocessing: Cloud masking, atmospheric correction

  2. Feature Extraction: Spectral bands, indices (NDVI, NDWI)

  3. Training Data: Collect representative samples for each class

  4. Model Training: Train classifier on training data

  5. Classification: Apply model to entire scene

  6. Validation: Assess accuracy with independent test data

Why Random Forest for Earth Observation?¬∂

Random Forest is one of the most popular algorithms for land cover classification because:

  • Handles high-dimensional data: Works well with many spectral bands (Sentinel-2 has 13 bands)

  • Robust to overfitting: Ensemble approach reduces variance

  • Feature importance: Reveals which bands are most informative

  • No feature scaling required: Unlike neural networks

  • Fast training: Efficient even with large datasets

  • Interpretable: Can visualize decision rules\

Random Forest: The ‚ÄúForest‚Äù Analogy¬∂

One tree (Decision Tree): One expert‚Äôs opinion

  • Can be very confident but sometimes wrong

  • Might overfit to specific training examples

Forest (Random Forest): Committee of experts

  • Each expert sees slightly different data

  • Each expert considers different features

  • Final decision: Majority vote

  • Wisdom of crowds: Group decision more reliable than individual

üí° If you ask 100 independent experts and 75 say ‚ÄúForest‚Äù, you can be more confident than if only 1 expert says ‚ÄúForest‚Äù.

C. Sentinel-2 Data Acquisition¬∂

This code block filters the Sentinel-2 Harmonized collection for low-cloud imagery within the 2024 date range, applies cloud masking, and generates a median composite clipped to Palawan for a clear, seamless RGB visualization.

# Define date range (using 2024 for better availability)
start_date = '2024-01-01'
end_date = '2024-12-31'

# Cloud masking function
def mask_s2_clouds(image):
    qa = image.select('QA60')
    cloud_bit_mask = 1 << 10
    cirrus_bit_mask = 1 << 11
    mask = (qa.bitwiseAnd(cloud_bit_mask).eq(0)
            .And(qa.bitwiseAnd(cirrus_bit_mask).eq(0)))
    return image.updateMask(mask).divide(10000)

print(f"Date range: {start_date} to {end_date}")
# Load Sentinel-2 collection
s2_collection = (ee.ImageCollection('COPERNICUS/S2_SR_HARMONIZED')
                 .filterBounds(palawan)
                 .filterDate(start_date, end_date)
                 .filter(ee.Filter.lt('CLOUDY_PIXEL_PERCENTAGE', 20))
                 .map(mask_s2_clouds))

# Create median composite
s2_composite = s2_collection.median().clip(palawan)

# Select bands
bands = ['B2', 'B3', 'B4', 'B8', 'B11', 'B12']
s2_composite = s2_composite.select(bands)

print(f"‚úì Sentinel-2 composite created")
print(f"Images in collection: {s2_collection.size().getInfo()}")
# Visualize RGB composite
rgb_vis = {'bands': ['B4', 'B3', 'B2'], 'min': 0.0, 'max': 0.3, 'gamma': 1.4}
Map.addLayer(s2_composite, rgb_vis, 'Sentinel-2 RGB')
Map

D. Calculate Spectral Indices¬∂

This code block calculates four key spectral indices‚Äî

  1. NDVI (vegetation),

  2. NDWI (water),

  3. NDBI (built-up areas), and

  4. EVI (enhanced vegetation)‚Äî

by performing mathematical transformations on specific Sentinel-2 bands to highlight different environmental features.

# Calculate all indices
ndvi = s2_composite.normalizedDifference(['B8', 'B4']).rename('NDVI')
ndwi = s2_composite.normalizedDifference(['B3', 'B8']).rename('NDWI')
ndbi = s2_composite.normalizedDifference(['B11', 'B8']).rename('NDBI')

evi = s2_composite.expression(
    '2.5 * ((NIR - RED) / (NIR + 6 * RED - 7.5 * BLUE + 1))',
    {
        'NIR': s2_composite.select('B8'),
        'RED': s2_composite.select('B4'),
        'BLUE': s2_composite.select('B2')
    }
).rename('EVI')

print("‚úì Spectral indices calculated")

E. Feature Stack Preparation¬∂

This code block combines the original Sentinel-2 spectral bands with the four (4) newly calculated indices into a single multi-band image stack, creating a comprehensive feature set for spatial analysis and machine learning classification.

# Create feature stack
feature_stack = (s2_composite
                 .addBands(ndvi)
                 .addBands(ndwi)
                 .addBands(ndbi)
                 .addBands(evi))

feature_names = feature_stack.bandNames().getInfo()
print(f"‚úì Feature stack created with {len(feature_names)} features")
print(f"Features: {feature_names}")

G. Random Forest Training¬∂

This code block configures and trains a Random Forest classifier using 100 decision trees to learn the relationship between the input spectral features and the corresponding land cover training labels.

Parameters¬∂

  1. n_trees

  • Description: Number of trees in forest

  • Typical values: 50-500

  • Impact: More trees ‚Üí better performance (diminishing returns)

  1. max_depth

  • Description: Maximum depth of each tree

  • Typical values: 10-50 or None

  • Impact: Deeper ‚Üí more complex, risk overfitting

  1. min_samples_split

  • Description: Min samples to split node

  • Typical values: 2-10

  • Impact: Higher ‚Üí simpler trees, less overfitting

  1. max_features

  • Description: Features per split

  • Typical values: ùëõ‚àö or log2(ùëõ)

  • Impact: Balance between accuracy and diversity

  1. bootstrap

  • Description: Use bootstrap sampling

  • Typical values: True

  • Impact: Almost always True for RF

üí° GEE‚Äôs ee.Classifier.smileRandomForest() defaults:

  • numberOfTrees: 100

  • variablesPerSplit: ùëõ‚àö (automatic)

  • minLeafPopulation: 1

# Train Random Forest classifier
rf_classifier = ee.Classifier.smileRandomForest(
    numberOfTrees=100,                            # The number of decision trees to create.
    variablesPerSplit=None,                       # The number of variables per split. If unspecified, uses the ‚àö of the number of variables.
    minLeafPopulation=1,                          # Only create nodes whose training set contains at least this many points
    bagFraction=0.632,                            # The fraction of input to bag per tree.
    seed=42                                       # The randomization seed
)

trained_classifier = rf_classifier.train(
    features=training_samples,
    classProperty='landcover',
    inputProperties=feature_names
)

print("‚úì Random Forest training complete")

I. Accuracy Assessment¬∂

This code block evaluates the model‚Äôs performance by generating an independent validation dataset and comparing the predicted land cover classes against known reference values. It calculates a confusion matrix to derive the Overall Accuracy and Kappa Coefficient, providing a statistical measure of how reliably the classifier identifies different land cover types.

Overall Accuracy

  • Definition: Percentage of correctly classified samples

  • Formula: Overall Accuracy = Correct Predictions / Total Samples

  • Interpretation: Simple, intuitive metric

  • Limitation: Can be misleading with imbalanced classes

  • Example: 95% accuracy sounds great, but if 95% of pixels are forest, a ‚Äúclassify everything as forest‚Äù model achieves 95%!

Kappa Coefficient

  • Definition: Measure of agreement beyond chance

  • Interpretation: Œ∫ < 0.4: Poor agreement Œ∫ = 0.4-0.6: Moderate agreement Œ∫ = 0.6-0.8: Good agreement Œ∫ > 0.8: Excellent agreement

  • Why use Kappa? Accounts for agreement by random chance, especially important for imbalanced datasets.

# Create validation dataset
validation_points = worldcover_remapped.stratifiedSample(
    numPoints=100,
    classBand='landcover',
    region=palawan,
    scale=30,
    seed=123,  # Different seed
    geometries=True
).filter(ee.Filter.neq('landcover', 0))

validation_samples = feature_stack.sampleRegions(
    collection=validation_points,
    properties=['landcover'],
    scale=10,
    geometries=False
)

# Classify validation set
validation_classified = validation_samples.classify(trained_classifier)

# Generate confusion matrix
confusion_matrix = validation_classified.errorMatrix('landcover', 'classification')
overall_accuracy = confusion_matrix.accuracy().getInfo()
kappa = confusion_matrix.kappa().getInfo()

print(f"Overall Accuracy: {overall_accuracy*100:.2f}%")
print(f"Kappa Coefficient: {kappa:.4f}")

Summary and Key Takeaways¬∂

Random Forest Ensemble¬∂

  • Combines many trees to reduce variance and improve stability

  • Uses bootstrap sampling (bagging) for training diversity

  • Uses random feature selection to decorrelate trees

  • Final prediction by majority voting (classification) or averaging (regression)

  • More robust than single trees, less prone to overfitting

Accuracy Metrics¬∂

  • Overall accuracy can hide class-specific problems

  • Other accuracy metrics includes: (1) Precision (user‚Äôs accuracy): Reliability of positive predictions, (2) Recall (producer‚Äôs accuracy): Completeness of detection, and (3) F1-score: Harmonic mean balancing precision and recall

  • Choice of metric depends on application cost (false positives vs. false negatives)

For Earth Observation¬∂

  • Random Forest works well with multi-spectral data

  • Feature importance reveals spectral signature insights

  • Confusion patterns often reflect spectral similarity

  • Fast training enables rapid iteration
